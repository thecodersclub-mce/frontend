Front-end Development - How the Web Works

This section outlines the foundational client-server interaction that allows websites to be accessed and displayed.

1. Web Browser

A software application (like Chrome, Firefox, or Safari) used by the client to access and display websites.

  • It sends requests to servers.
  • It renders the received content (HTML, CSS, JavaScript) into a visual web page.

2. Web Server

A computer system that stores and serves website files (HTML documents, images, videos, etc.).

  • It stores the website's resources.
  • It responds to browser requests by sending back the required resources.

3. Request & Response Cycle

This is the communication protocol between the browser (client) and the server.

  1. Request: The browser sends an HTTP request to a server (e.g., a GET request to load a page or a POST request to submit data).
  2. Processing: The server processes the request, locates the necessary files or data.
  3. Response: The server sends an HTTP response back to the browser, containing the requested resource (e.g., an HTML file) and a status code.
  4. Rendering: The browser reads the HTML and associated files and displays the web page to the user.

The entire process ensures that users can interact with resources stored remotely on servers.

Front-end Development - Basic HTML Structure

1. What is HTML?

HTML stands for HyperText Markup Language. It's the standard language used to create webpages.

Every webpage has a basic skeleton, made of HTML tags that tell the browser how to display content.

2. Basic HTML Skeleton

Here is the simplest complete structure of an HTML document:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Webpage</title>
  </head>
  <body>
    <h1>Welcome to My Website!</h1>
    <p>This is my first HTML page.</p>
  </body>
</html>

3. Explanation of Each Part

Part Description
<!DOCTYPE html> Tells the browser that this is an HTML5 document.
<html> The root element of the webpage. All other content and tags are placed inside this tag.
<head> Contains meta information (not visible on the page), like the title, character set, and links to styles or scripts.
<title> The text that appears on the browser tab.
<body> Contains everything you see on the webpage—text, images, links, buttons, etc.
<h1> A heading tag. Headings range from <h1> (largest) to <h6> (smallest).
<p> A paragraph tag for standard blocks of text.

Front-end Development - CSS Selectors Overview

CSS selectors are patterns used to target specific HTML elements to apply styles to them. They determine which elements a CSS rule will affect.

Common Selector Types

Selector Type Pattern Purpose & Example
Universal * Selects all elements on the page.
Example: * { margin: 0; }
Type (Element) p, h1, div Selects all elements of a **specific HTML tag**.
Example: p { color: blue; }
Class .classname Selects all elements with a specific class attribute value. (Reusable)
Example: .highlight { background-color: yellow; }
ID #idname Selects a single, unique element with a specific id attribute value.
Example: #header { font-size: 24px; }
Attribute [attribute] Selects elements based on the presence of an attribute or its exact value.
Example: input[type="submit"] { background-color: green; }

Combinators (Combining Selectors)

Combinators are used to link multiple simple selectors to create more specific targeting rules:

  • Descendant Combinator (Space): div p
    • Selects all <p> elements that are anywhere inside a <div> element.
  • Child Combinator (>): ul > li
    • Selects all <li> elements that are a direct child of a <ul> element.

Responsive Web Design – Mobile-First Approach:

Responsive Web Design (RWD) ensures a website looks and works perfectly on all devices — mobiles, tablets, and desktops.

The Mobile-First Approach is a key strategy in RWD, meaning you design and code for smaller screens first, then gradually enhance the layout and features for larger devices.

Key Points:

  • ✓ Start with simple, fast-loading mobile layouts, focusing on core content and essential functionality.
  • ✓ Use fluid grids (like Flexbox or Grid), flexible images (using max-width: 100%;), and CSS media queries for breakpoints.
  • ✓ Prioritize content and usability specifically for mobile users, where screen space is limited.
  • ✓ Scale up for bigger screens using progressive enhancement, adding complexity and extra elements only as screen space allows.

Goal: Deliver a seamless user experience across all screen sizes, ensuring your site is accessible and functional everywhere.

Topic: Mastering Flexbox Layout

Hey Coders! Today’s puzzle is all about one of the most powerful CSS tools — **Flexbox**! It’s designed to provide an efficient way to lay out, align, and distribute space among items in a container, even when their size is unknown or dynamic.

Can you piece together how these properties fit to create perfect layouts?

The Flexbox Puzzle Pieces (Container Properties):

These properties are applied to the parent container (the element with display: flex;) and control the overall layout and alignment of its children (flex items):

  • display: flex; → The starting point — turns your container into a Flexbox world!
  • flex-direction → Decides the flow of items along the Main Axis: row (default), column, row-reverse, or column-reverse.
  • justify-content → Aligns your items along the Main Axis (e.g., flex-start, center, space-between, space-around).
  • align-items → Aligns items along the Cross Axis (perpendicular to the main axis, e.g., flex-start, center, stretch).
  • flex-wrap → Tells items whether to stay in one line (nowrap) or **wrap** to the next line (wrap) if they exceed the container width.
  • align-content → Controls the space between rows (or columns) when wrapping happens (i.e., when flex-wrap is active).
  • gap (or row-gap/column-gap) → Adds perfect spacing between items — a modern replacement for using external margins!

CSS Grid: The Modern Layout Superpower!

Design clean, flexible, and fully responsive layouts with just a few lines of CSS. No more messy floats or confusing complex Flexbox setups — CSS Grid gives you total control over two-dimensional layouts!

With CSS Grid, you can:

  • Create complex layouts easily (rows and columns simultaneously).
  • Align and space elements perfectly within a grid structure.
  • Make your site look great on any device (inherently responsive).
  • Mix with Flexbox for even more power (Grid handles the macro-layout, Flexbox handles the micro-alignment within cells).

Perfect for:

  • Responsive websites
  • Dashboards
  • Portfolios
  • Modern web apps

Example Concept:

.container {
    display: grid;
    /* Defines 3 columns: two flexible, one fixed */
    grid-template-columns: 1fr 1fr 200px; 
    /* Defines two rows with flexible heights */
    grid-template-rows: auto 1fr; 
    gap: 10px;
}
Tip: If you’re building your portfolio, try designing it using CSS Grid — you’ll be amazed at how clean and professional it looks!

CSS Variables, officially known as Custom Properties, allow you to store specific values to be reused throughout your stylesheet. Think of them as variables in a programming language—you define a value once and use it everywhere.

Why use them?

  • Maintainability: Need to change your site's primary color? Just update one variable instead of hunting for every instance of that color code across hundreds of lines.
  • Readability: Code like color: var(--primary-color); is much clearer than color: #4A90E2;.
  • Theming: They are the cornerstone of dynamic features like dark/light mode switches or switching between color palettes.

Syntax

You declare a variable using a double-hyphen prefix (--) and access it using the var() function. It's best practice to declare global variables in the :root pseudo-class.

/* Declaration (Global Scope) */
:root {
  --primary-color: #007bff;
  --spacing-unit: 1rem;
}

/* Usage */
.header {
  background-color: var(--primary-color);
  padding: var(--spacing-unit);
}

Example: A Simple Themed Button

Here, we define a primary color and use it for a button. Changing the --primary-color value in :root will update the button instantly.

/* CSS */
:root {
  --primary-color: #4A90E2; /* Blue */
  --text-color: white;
}

.themed-button {
  background: var(--primary-color);
  color: var(--text-color);
  padding: 10px 20px;
  border: none;
  border-radius: 5px;
}

/* You can easily create a 'success' theme that overrides the color locally: */
.themed-button.success {
  --primary-color: #5cb85c; /* Green, overrides local scope */
}

JS Basics: Variables, Data Types, and Operators

Variables:

Variables are used to store data values. The choice of keyword affects the scope and mutability of the variable: 

var name = "John"; // old way, function-scoped
let age = 25; // modern way, block-scoped, can be reassigned
const PI = 3.14; // constant, block-scoped, cannot be reassigned

Data Types:

Data types classify the kind of data a variable holds.

Primitive Types (Single, immutable values):
  1. String – text data (e.g., "Hello")
  2. Number – integers and floating-point numbers (e.g., 10, 3.14)
  3. Boolean – logical values (e.g., true, false)
  4. Undefined – variable declared but not yet assigned a value
  5. Null – intentional absence of any object value
  6. Symbol – unique identifier (introduced in ES6)
  7. BigInt – used for integers larger than $2^{53}-1$
Non-Primitive Types (Collection of values/more complex):
  1. Object (includes arrays, functions, and standard objects)
let person = {name: "Alice", age: 22}; // Object
let fruits = ["apple", "mango"]; // Array (a type of Object)

Operators:

Operators perform operations on variables and values.

  1. Arithmetic: +, -, *, /, % (modulo), ** (exponentiation)
  2. Assignment: =, +=, -=, *=, etc.
  3. Comparison: ==, === (strict equality), !=, >, <, >=, <=
  4. Logical: && (AND), || (OR), ! (NOT)
  5. String: + (concatenation)
  6. Ternary: condition ? value1 : value2 (Shorthand for if/else)
Example:
let x = 10, y = 5;
let sum = x + y;            // 15 (Arithmetic)
let isEqual = x === y;      // false (Comparison)
let message = (x > y) ? "Yes" : "No"; // "Yes" (Ternary)

What is DOM?

DOM (Document Object Model) is a tree-like structure created by the browser for every webpage. It represents all the HTML elements as objects, allowing JavaScript to access, change, or delete them.

Why it’s Important

  • Makes web pages dynamic and interactive.
  • Allows content updates without reloading the page.

Modern Javascript (ES6): Let, Const, and Arrow Functions:

1. let

  • Used to declare block-scoped variables (only accessible within `{ }`).
  • Can be reassigned, but **not redeclared** in the same scope.
Example:
let count = 10;
count = 20; // allowed
// let count = 30; // NOT allowed (redeclaration)

2. const

  • Declares constant variables (the identifier cannot be reassigned).
  • Also block-scoped.
  • Note: For objects and arrays, the content can still be modified, but the variable itself cannot be pointed to a new object/array.
Example:
const PI = 3.14;
// PI = 3.1415; // NOT allowed (reassignment)

const settings = { theme: 'dark' };
settings.theme = 'light'; // Allowed (modifying content)

3. Arrow Functions (=>)

  • Provides a shorter, cleaner syntax for writing functions.
  • Crucially, they do not bind their own this value, inheriting it from the surrounding code (lexical this).
Example:
// Concise function for addition
const add = (a, b) => a + b;
console.log(add(2, 3)); // 5

// Function with single parameter (parentheses optional)
const double = num => num * 2;

In short:

  • let → flexible, block-scoped, reassignable.
  • const → fixed, block-scoped, immutable identifier.
  • => → concise, modern way to write functions with lexical this.

Destructuring, Spread, and Template Literals in JavaScript:

1. Destructuring

Destructuring allows you to unpack values from arrays or properties from objects into separate, readable variables.

  • Use Case: Simplifies variable assignment and improves code readability.
Example:
// Array Destructuring
const numbers = [10, 20, 30];
const [a, b, c] = numbers;
console.log(a); // 10

// Object Destructuring
const user = { name: "Bala", age: 21 };
const { name, age } = user;
console.log(name); // Bala

2. Spread Operator (...)

The Spread Operator allows an iterable (like an array or object) to be expanded into its individual elements or key-value pairs.

  • Use Case: Useful for copying, merging, and passing data dynamically (e.g., merging two arrays or adding properties to an object).
Example:
// Array Spread (Merging)
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5];
console.log(arr2); // [1, 2, 3, 4, 5]

// Object Spread (Copying and Extending)
const obj1 = { a: 1, b: 2 };
const obj2 = { ...obj1, c: 3 };
console.log(obj2); // { a: 1, b: 2, c: 3 }

3. Template Literals

Template literals allow embedding variables and expressions inside strings using backticks (`` ` ``) and ${} placeholders.

  • Use Case: Makes string concatenation cleaner, more readable, and flexible, especially for multi-line strings.
Example:
const name = "Bala";
const age = 21;
console.log(`My name is ${name} and I am ${age} years old.`);
// Output: My name is Bala and I am 21 years old.

Async JavaScript: Promises & Async/Await:

In JavaScript, not everything happens instantly — some tasks (like fetching data or loading APIs) take time. That's where asynchronous programming comes in!

Promises:

  • Purpose: Handle tasks that take time and let you run code only after they are successfully completed or have failed.
  • States: A Promise can be Pending (waiting), Fulfilled (successful), or Rejected (failed).
  • Example: Fetching data from a server using .then() and .catch() to handle success and failure.

Async/Await:

  • Purpose: A cleaner, modern way to write Promise-based asynchronous code.
  • Benefit: It allows you to write asynchronous code that *looks* synchronous, making it easier to read and debug, while still running efficiently in the background.

Syntax Example:

async function getData() {
  try {
    const response = await fetch(url);
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Fetch failed:', error);
  }
}

Why Use Async JS?

  • Load data smoothly without freezing the user interface (non-blocking).
  • Handle multiple tasks at once efficiently.
  • Make your apps faster and user-friendly.

    <body>
    
        <div id="root"></div> <!-- Mount point for the React App -->
    
        <script type="text/babel">
            
            // --- 1. Utility Function ---
            const getInitialTasks = () => {
                const storedTasks = localStorage.getItem('react-todo-list');
                if (storedTasks) {
                    return JSON.parse(storedTasks);
                }
                return [];
            };
    
            // --- 2. TodoItem Component ---
            const TodoItem = ({ task, toggleComplete, deleteTask }) => (
                <li className={`todo-item ${task.isCompleted ? 'completed' : ''}`}>
                    <span className="todo-text">{task.text}</span>
                    <div>
                        <button 
                            className="toggle-btn" 
                            onClick={() => toggleComplete(task.id)}
                            title={task.isCompleted ? "Mark as Incomplete" : "Mark as Complete"}
                        >
                            {task.isCompleted ? '🔄' : '✅'}
                        </button>
                        <button 
                            className="delete-btn" 
                            onClick={() => deleteTask(task.id)}
                            title="Delete Task"
                        >
                            🗑️
                        </button>
                    </div>
                </li>
            );
    
            // --- 3. TodoApp Component (Main Logic) ---
            const TodoApp = () => {
                const [tasks, setTasks] = React.useState(getInitialTasks);
                const [taskInput, setTaskInput] = React.useState('');
    
                // EFFECT: Save tasks to Local Storage whenever the 'tasks' state changes
                React.useEffect(() => {
                    localStorage.setItem('react-todo-list', JSON.stringify(tasks));
                }, [tasks]);
    
                // Handler to add a new task (CREATE)
                const addTask = (e) => {
                    e.preventDefault();
                    const trimmedInput = taskInput.trim();
    
                    if (trimmedInput) {
                        const newTodo = {
                            id: Date.now(),
                            text: trimmedInput,
                            isCompleted: false,
                        };
                        setTasks([...tasks, newTodo]);
                        setTaskInput('');
                    }
                };
    
                // Handler to toggle a task's completion status (UPDATE)
                const toggleComplete = (id) => {
                    setTasks(
                        tasks.map(task => 
                            task.id === id ? { ...task, isCompleted: !task.isCompleted } : task
                        )
                    );
                };
    
                // Handler to delete a task (DELETE)
                const deleteTask = (id) => {
                    setTasks(tasks.filter(task => task.id !== id));
                };
    
                return (
                    <div className="todo-container">
                        <h1>Interactive To-Do List</h1>
                        
                        {/* Input Form */}
                        <form className="todo-form" onSubmit={addTask}>
                            <input
                                type="text"
                                placeholder="Add a new task..."
                                value={taskInput}
                                onChange={(e) => setTaskInput(e.target.value)}
                            />
                            <button type="submit">Add</button>
                        </form>
                        
                        {/* Task List */}
                        {tasks.length === 0 ? (
                            <p>No tasks yet! Add one above.</p>
                        ) : (
                            <ul className="todo-list">
                                {tasks.map(task => (
                                    <TodoItem
                                        key={task.id}
                                        task={task}
                                        toggleComplete={toggleComplete}
                                        deleteTask={deleteTask}
                                    />
                                ))}
                            </ul>
                        )}
                    </div>
                );
            };
    
            // --- 4. Render the App ---
            const container = document.getElementById('root');
            const root = ReactDOM.createRoot(container);
            root.render(<TodoApp />);
            
        </script>
    </body>
